| Property | Value |

## Spacing Scale

Standard Tailwind 4-pt scale is active. Declared values used in this phase:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline spinner margin |
| sm | 8px | Gap between spinner and button text (not applicable — spinner replaces text). Gap between empty state heading and body copy. |
| md | 16px | Toast internal padding (px-4 py-4), button horizontal padding baseline |
| lg | 24px | Toast stack gap |
| xl | 32px | Toast container bottom offset from viewport edge |
| 2xl | 48px | Toast container right offset from viewport edge |

Exceptions:
- Toast container positioned `bottom-6 right-6` (24px) on mobile (max-[480px]), `bottom-8 right-8` (32px) on desktop. Mobile reduces both axes equally to avoid obscuring the floating sidebar pill which lives bottom-right.
- ActionButton minimum touch target: 44px height on mobile (via `min-h-[44px]` on `max-[480px]`). Buttons that are naturally taller (py-3.5) already satisfy this — enforce only on compact pill buttons.

## Component Contracts

### ToastContainer.vue

**Placement:** Fixed, `bottom-6 right-6` on mobile (max-[480px]), `bottom-8 right-8` on desktop. z-index above sidebar scrim (`z-50` minimum). Stack direction: new toasts append above existing ones (translate upward).

**Toast width:** `w-[320px] max-w-[calc(100vw-48px)]` — never clips on narrow phones.

**Toast anatomy:**
```
┌──────────────────────────────────────────┐
│ ▌  Message text (14px/400)               │  ← 3px left stripe (clay or failed)
│    [Fortryd]                  [×]        │  ← undo link + dismiss icon (undo type only)
└──────────────────────────────────────────┘
```

- Background: `bg-paper` (#f8f3e9)
- Border: `border border-line` (rgba 26,20,16 / 0.10) + `rounded-md`
- Shadow: `shadow-card` (0 14px 30px rgba(26,20,16,0.10))
- Left stripe: 3px solid, `clay` for success, `status.failed` (#a85a48) for error/undo
- Message: 14px/400, `text-ink` (#1a1410)
- Dismiss button (×): `Icon name="close"` size 14, `text-ink-mute`, hover `text-ink`, `aria-label="Luk besked"`
- Undo link: 13px/500 medium, `text-clay-deep` (#8b4e2c), underline on hover

**Toast types and auto-dismiss:**

| Type | Left stripe | Auto-dismiss | Persist |
|------|-------------|--------------|---------|
| success | clay (#b87a4e) | 3500ms | No |
| error | status.failed (#a85a48) | No | Until dismissed |
| undo | status.failed (#a85a48) | 5000ms | No (undo window = 5s) |

**Animation:**
- Enter: slide up 8px + fade in. Duration 200ms ease-out. CSS: `translate-y-2 opacity-0` → `translate-y-0 opacity-100`.
- Exit: fade out only. Duration 150ms ease-in. `opacity-0`.
- Respects `prefers-reduced-motion`: skip translate, keep opacity-only fade (existing pattern in main.css).

**Stacking:** Maximum 3 toasts visible simultaneously. If a 4th arrives, the oldest is removed immediately (no animation).

**useToast() composable API:**

```ts
const { toast } = useToast()

toast.success('Ændringer gemt')
toast.error('Noget gik galt — prøv igen')
toast.undo('Ret slettet', { onUndo: () => restore() })
```

Composable provides reactive `toasts` array consumed by `ToastContainer.vue`. Each toast: `{ id, type, message, undoLabel?, onUndo?, createdAt }`.

**Mount point:** Inside `<platform>` layout only. `public.vue` layout does not get ToastContainer (public pages are read-only for diners, no actions to confirm).

### ActionButton.vue

**Purpose:** Any button that triggers an API call. Replaces ad-hoc `saving ? 'Saving...' : 'Gem ændringer'` patterns across platform pages.

**Props:**
```ts
interface ActionButtonProps {
  loading?: boolean        // shows spinner, disables interaction
  disabled?: boolean       // explicit disable independent of loading
  variant?: 'primary' | 'ghost' | 'danger'   // maps to existing CSS classes
  type?: 'button' | 'submit'
  size?: 'md' | 'sm'      // md = default top-btn sizing, sm = compact pill
}
```

**Spinner spec:**
- Inline SVG circle arc, 14×14px
- Stroke: currentColor (inherits button text color)
- Stroke-width: 1.6
- Animation: `rotate-slow` keyframe already defined in tailwind.config.ts (14s linear infinite) — use a faster variant: `animate-[spin_0.7s_linear_infinite]` (standard Tailwind spin utility)
- Spinner replaces slot content entirely while `loading=true` — no text alongside spinner
- No layout shift: button width is preserved via `min-w` equal to natural content width, or fixed width classes. Prefer `w-full` on form submit buttons (already full-width). For top-bar pill buttons use `min-w-[100px]`.

**States:**

| State | Visual |
|-------|--------|
| Default | Slot content (label text) |
| Loading | Spinner (14px spin), button `disabled`, `opacity-70`, cursor-not-allowed |
| Disabled (not loading) | `opacity-50`, cursor-not-allowed |

**Variants map to existing CSS:**

| Variant | Class basis |
|---------|-------------|
| primary | `top-btn top-btn--primary` |
| ghost | `top-btn` |
| danger | `top-btn` + `text-[#a85a48] border-[rgba(168,90,72,0.30)]` hover `bg-[rgba(168,90,72,0.06)]` |

### error.vue (app/error.vue)

**Scope:** Nuxt error page — catches all unhandled 404, 500, and unhandled runtime errors.

**Layout:** Centered single column. Full viewport height (`min-h-screen`). Background: `bg` (#f3ede2) with existing body gradient from main.css (automatically applied). No sidebar, no topbar.

**Content structure:**
```
[LogoMark — 32px height]
[gap 48px]
[Error code in mono eyebrow style — e.g. "404"]   ← optional, shown only if statusCode present
[gap 16px]
[Heading — display font, ~32px]
[gap 12px]
[Body — 15px/400, ink-mute]
[gap 32px]
[CTA button]
```

**Single layout for all error codes** (D-09 locked decision): no contextual differentiation between 404 and 500.
